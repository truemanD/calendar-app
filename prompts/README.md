# Система промптов для ролей разработки

Эта директория содержит промпты для различных ролей в процессе разработки ПО. Каждая роль работает изолированно и сохраняет результаты своей работы в отдельную директорию проекта.

## Структура ролей

1. **Владелец продукта** (`product-owner.md`) - видение продукта, анализ рынка, метрики, гипотезы, конкуренты
2. **Бизнес-аналитик** (`business-analyst.md`) - анализ требований, создание спецификаций
3. **Архитектор** (`architect.md`) - проектирование архитектуры системы
4. **UI/UX дизайнер** (`ui-ux-designer.md`) - проектирование интерфейсов и пользовательского опыта
5. **Разработчик фронтенда** (`frontend-developer.md`) - реализация клиентской части
6. **Разработчик бэкенда** (`backend-developer.md`) - реализация серверной части
7. **Тестировщик** (`qa-engineer.md`) - тестирование и обеспечение качества
8. **DevOps инженер** (`devops-engineer.md`) - инфраструктура, CI/CD, деплоймент

## Принципы работы

- **Изоляция**: Каждая роль работает независимо и не вмешивается в работу других ролей
- **Структурированное хранение**: Все артефакты сохраняются в соответствующие директории проекта
- **Документирование**: Все решения и артефакты должны быть задокументированы
- **Стандартизация**: Использование единых форматов и соглашений в рамках роли

## Использование

Каждый промпт можно использовать как инструкцию для AI-агента соответствующей роли. Промпт содержит:
- Описание роли и обязанностей
- Правила работы и изоляции
- Структуру директорий для сохранения результатов
- Форматы документов и артефактов
- Примеры работы
